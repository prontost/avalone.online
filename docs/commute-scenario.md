# Commute scenario — поездки на работу

> Первый реализуемый сценарий Avalone.
> Цель: убрать хаос координации развозки временных работников.

## Цель сценария

Сделать так, чтобы поездка на работу и с работы не требовала постоянных звонков, поисков по цехам и неопределённости.

Конкретный результат:
- Вечером человек знает, когда и с кем едет утром.
- Утром он знает, где и во сколько его заберут.
- В течение дня он видит актуальный статус: кто уже уехал, кто остался, есть ли свободные места.
- Вечером он быстро находит свою машину и уезжает домой.

## Акторы и роли

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ Работодатель │────▶│  Самоним    │────▶│   Водитель  │
│   (завод)    │     │ (dispatch)  │     │  (водит)    │
└─────────────┘     └─────────────┘     └──────┬──────┘
                                               │
                                               ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ Распределяющий│◀───│  Пассажир   │◀────│   Машина    │
│   на заводе   │     │  (едет)     │     │  (маршрут)  │
└─────────────┘     └─────────────┘     └─────────────┘
```

### Работодатель (завод)
- Даёт заявку самониму.
- Указывает количество людей, требования, время, условия.
- Иногда принимает/отправляет люней домой раньше.

### Самоним (диспетчер / dispatch agency)
- Получает заявки от работодателей.
- Формирует список людей на объект.
- Распределяет людей по машинам.
- Информирует водителей и пассажиров.

### Водитель
- Имеет машину и права.
- Забирает пассажиров по маршруту.
- Довозит до завода.
- Вечером развозит/забирает с завода.

### Пассажир
- Нуждается в доставке на работу.
- Может иметь предпочтения: с кем ездить, во сколько, откуда.

### Распределяющий на заводе
- Знает особенности цехов.
- Распределяет людей по участкам.
- Может влиять на то, кто остаётся на переработку / уходит раньше.

### Машина / маршрут
- Транспортная единица.
- Имеет вместимость, точки сбора, время в пути.

## Happy path

### Вечером (или накануне)

1. Самоним получает заявку от завода.
2. Самоним формирует список людей на объект.
3. Система автоматически предлагает оптимальное распределение по машинам:
   - учитывает вместимость;
   - учитывает локации;
   - учитывает предпочтения пассажиров;
   - учитывает надежность водителей.
4. Самоним подтверждает или корректирует распределение.
5. Каждый пассажир получает уведомление:
   - время выезда;
   - точка сбора;
   - водитель;
   - контакт;
   - fellow passengers.
6. Водитель получает маршрут и список пассажиров.

### Утром

7. Водитель отмечает, что выехал.
8. Пассажиры видят приближение машины / статус.
9. Водитель забирает людей по порядку.
10. Все приезжают на завод вовремя.

### В течение дня

11. Система получает обновления от распределяющего:
    - кто отправлен домой раньше;
    - кто остался на переработку;
    - кто на каком участке.
12. Водители и пассажиры видят актуальный статус возврата.

### Вечером

13. Система формирует список уезжающих.
14. Люди получают уведомления: где и во сколько их заберут.
15. Водитель формирует маршрут и развозит людей.
16. Все добираются домой без хаоса.

## Болевые сценарии (как сейчас)

### Боль 1: неизвестно, кто на какой машине
- Распределение происходит в голове самонима или в разговорах.
- Пассажиры узнают об этом поздно или не полностью.

### Боль 2: неизвестно, во сколько выезжать
- Время сбора определяется по телефону или в чате.
- Часто передвигается.

### Боль 3: водитель не знает, где кого забирать
- Адреса или точки сбора передаются устно.
- Водитель может заблудиться или опоздать.

### Боль 4: кто-то закончил раньше
- Человек готов уезжать, но не знает, есть ли свободное место.
- Звонит водителю / самониму / другим пассажирам.

### Боль 5: кто-то остался на переработку
- Машина уезжает без него.
- Он не знает, когда следующая машина или как добраться.

### Боль 6: машина полна / не хватает мест
- Внезапно нужно везти больше людей, чем вмещает машина.
- Решается на месте через звонки.

### Боль 7: водитель не вышел / машина сломалась
- Все сидят и ждут, пока самоним найдёт замену.
- Нет резервного плана в системе.

### Боль 8: языковые барьеры
- Люди разных национальностей.
- Информация теряется при переводе.

### Боль 9: нет единой правды
- Каждый знает только свой кусок.
- Общая картина собирается через десятки сообщений и звонков.

## Текущие инструменты

- **KakaoTalk группы** — основной канал коммуникации.
- **Звонки** — когда нужно срочно что-то выяснить.
- **Листочки / заметки у самонима** — распределение людей по машинам.
- **Word-of-mouth** — адреса, предпочтения, изменения.
- **Личные контакты** — каждый звонит своему знакомому водителю.

Нет центральной системы статуса. Всё держится на голове самонима и постоянной коммуникации.

## Гипотезы решений

### Гипотеза 1: Цифровая доска самонима
- Самоним видит все заявки, людей, машины.
- Система предлагает распределение.
- Один клик — и все получают уведомления.

### Гипотеза 2: Мобильный статус
- Каждый участник видит свой статус: куда едет, с кем, во сколько.
- Водитель видит маршрут.
- Все видят изменения в реальном времени.

### Гипотеза 3: Smart matching
- Учитывать локации, вместимость, предпочтения, историю.
- Предлагать оптимальные машины.
- Учитывать, что кто-то не хочет ездить с кем-то конкретным.

### Гипотеза 4: Обратная связь с завода
- Распределяющий или сам работник может отметить: "я закончил", "я на переработке", "мне нужно место".
- Система автоматически пересчитывает вечерние машины.

### Гипотеза 5: Резервные машины / водители
- Система знает, кто ещё может подвезти.
- При сбое предлагает альтернативу.

### Гипотеза 6: Многоязычность
- Интерфейс и уведомления на языках пользователей.
- Автоперевод для общих сообщений.

## Связь с метавселенной

Этот сценарий использует фундаментальные сущности Avalone:

| Сущность | Как проявляется в сценарии |
|----------|---------------------------|
| **Человек** | Работник, водитель, самоним, распределяющий. |
| **Организация** | Самущиль, завод. |
| **Место** | Дома пассажиров, точки сбора, завод, цех. |
| **Событие** | Утренняя смена, переработка, возвращение домой. |
| **Возможность** | Работа / смена на заводе. |
| **Транспорт** | Машина, маршрут. |
| **Репутация** | Надёжность водителя, пунктуальность пассажира, рейтинг самонима. |
| **Сообщество** | Группа людей, ездящих на один объект. |
| **Навык** | Водительские права, языки, опыт на конкретных участках. |
