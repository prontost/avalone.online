# Portal navigation — концепция входа в Avalone

> Главная страница портала — это не просто меню. Это вход в новую цифровую вселенную поверх реального мира.
> Пользователь должен видеть всё, что есть и что будет, и иметь кратчайший путь к своей текущей цели.

## Что должно быть на главной

Главная страница решает три задачи одновременно:

1. **Ориентация.** Пользователь сразу понимает, куда он попал и что здесь можно делать.
2. **Действие.** Один-два тапа до самых частых задач: поехать на работу, найти смену, посмотреть статус.
3. **Видение будущего.** Все разделы метавселенной видны и структурированы. Неразработанные разделы показаны как "coming soon" / "скоро" — без обмана, но с интригой.

## Принципы навигации

### 1. Thumb-zone first
- Основные действия — в нижней трети экрана.
- Нижняя навигация работает на 30-40% быстрее, чем боковое меню, для частых переключений.
- Первичных пунктов не больше 4-5, иначе когнитивная перегрузка.

### 2. Progressive disclosure
- Сначала показывать самое важное.
- Детали и вторичные опции раскрывать по тапу.
- Не бросать пользователя в лес вложенных меню сразу.

### 3. Кратчайший путь
- От старта до цели — минимум тапов.
- Частые действия доступны напрямую с главной.
- Контекстная навигация: если утро и есть смена — предлагать "поехать на работу".

### 4. Постоянная ориентация
- Пользователь всегда должен знать, где находится.
- Хлебные крошки, заголовки, индикаторы текущей ветки.

## Паттерны навигации

### Бургер-меню с множеством вложенностей
- Плюсы: вместительность, можно показать всю иерархию.
- Минусы: скрывает содержимое, увеличивает путь до цели, легко потеряться.
- Где использовать: для полного каталога всех веток и модулей, но не как основной способ.

### Горизонтальная иерархия
- Вместо углубления вниз — движение вбок.
- Верхний уровень: ветки (Работа, Жильё, Образование...).
- При выборе ветки — горизонтальный скролл модулей.
- Плюсы: быстрое переключение, наглядная карта мира.
- Минусы: не подходит для очень глубоких структур.

### Command palette / поиск
- Горячая клавиша или иконка поиска.
- Пользователь печатает "работа завтра" и сразу попадает в нужный модуль.
- Плюсы: кратчайший путь для опытных пользователей.
- Минусы: требует знания названий.

### Bottom navigation + contextual cards
- Нижняя панель: 4-5 главных веток.
- Главный экран: контекстные карточки под текущую ситуацию.
- Пример: "Сегодня вы едете на завод с Васей, 06:30".

### Floating Action Button (FAB)
- Круглая кнопка для главного действия в текущем контексте.
- Пример: "Найти машину", "Записаться на смену", "Предложить поездку".

### Bento grid / карточная сетка
- Главная как сетка карточек: работа, жильё, обучение, здоровье...
- Каждая карточка показывает статус и быстрые действия.
- Плюсы: визуально привлекательно, хорошо масштабируется.

## Визуальные фичи

### Glassmorphism & layered depth
- Полупрозрачные слои, blur, свечение.
- Создаёт ощущение пространства и глубины.
- Помогает выделить активный элемент.

### Spatial UI
- Элементы располагаются в пространстве, а не просто в списке.
- Пример: ветки метавселенной как "планеты" или "зоны" вокруг профиля пользователя.

### Micro-animations
- Плавные переходы между ветками.
- Обратная связь на действия.
- Не анимация ради анимации, а навигационная подсказка.

### Adaptive theming
- Интерфейс меняется под время суток, локацию, контекст.
- Утро — акцент на работу и передвижение.
- Вечер — акцент на возвращение и планирование завтра.

### Empty states как teaser
- Для неразработанных разделов — не пустой экран, а интерактивный тизер.
- Пример: "Образование — скоро. Узнай первым." + кнопка "сообщить, когда запустим".

## Coming soon / teaser

### Паттерн "держи руку на пульсе"
- Все ветки метавселенной видны.
- Разработанные — активны.
- Неразработанные — полупрозрачные, с пометкой "скоро".
- Пользователь видит масштаб и понимает, что платформа растёт.

### Примеры текстов
- "Работа — уже работает"
- "Образование — в разработке"
- "Туризм — скоро"
- "Здоровье — планируется"

### Подписка на запуск
- Для coming-soon разделов можно собирать заявки.
- Это даёт данные о спросе и канал коммуникации.

## Пример структуры главной

```
┌─────────────────────────────┐
│  Привет, Денис               │
│  Сегодня: едешь на завод     │
│  06:30, водитель Вася        │
├─────────────────────────────┤
│                             │
│  [Работа]  [Образование]    │
│  [Жильё]   [Туризм]         │
│  [Здоровье][Сообщества]     │
│                             │
├─────────────────────────────┤
│  Быстрые действия            │
│  [Найти машину] [Статус]    │
│  [Смена]        [Люди]      │
├─────────────────────────────┤
│  [🏠] [🔍] [➕] [💬] [👤]   │
└─────────────────────────────┘
```

## Пример альтернативной навигации: горизонтальная иерархия

```
Главная → Работа → Арбайт → Мои смены → Завтра
                → Фриланс
                → Развозка
        → Образование → Курсы
                      → Языки
        → Жильё → Аренда
                → Соседи
```

Вместо глубокого бургер-меню — горизонтальный скролл веток, затем горизонтальный скролл модулей.

## Рекомендации для MVP

1. Начать с **bottom navigation + contextual cards**.
2. Главная ветка "Работа" — полностью работающая.
3. Остальные ветки — в режиме teaser.
4. Добавить **command palette / поиск** для опытных пользователей.
5. Использовать **bento grid** для визуальной привлекательности.
6. Собирать метрики: какие разделы открывают, как часто, сколько тапов до цели.
